ICAROUS Configuration

There are several paramters that can be chosen by a user to fine-tune Icarous behaviors to a specific mission. These parameters are located in cFS/bin/ram/icarous.txt. All applications access this file to extract relevant parameters. A brief explanation of a few relevant parameter is provided below:

HTRESHOLD Horizontal buffer (m) added to geofence polygons. The polgons are expanded horizontally by this amount
VTHRESHOLD Vertical buffer (m) added to geofence polygons. The polygons are expanded vertically by this amount
HSTEPBACK (m) Horizontal bounce back distance on encountering a keep-in fence conflict.
HSTEPBACK (m) Vertical bounce back distance on encountering a keep-in fence conflict (only used in vertical violations).
CHEAP_SEARCH 1 (Use RRT algorithm), 0 (Use A* algorithm)
GRIDSIZE (m) Grid size used for discretization of x-y space in A* search
LOOKAHEAD (s) Look ahead horizon to check for geofence violations
RES_SPEED (m/s) Ground speed used for when executing conflict resolution maneuvers
XTRK_GAIN Proportional gain used by the path conformance controller
XTRK_DEV (m) Allowed cross track deviation
TRK_HEADING (degrees) Bearing with respect to leader
TRK_DISTH (m) Horizontal distance to leader
TRK_DISTV (m) Vertical distance to leader
DAA_CONFIG location of DAA configuration
CAPTURE_H (m) horizontal distance within which a waypoint is considered reached
CAPTURE_V (m) vertical distance within which a waypoint is considered reached
